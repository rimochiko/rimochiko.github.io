JS+TS数独练习小结

最近想再接触一点新的知识，于是就把数独游戏的慕课课程跟着做了一遍，这个小游戏最初是先用JS来实现，后面再改装成了typescript的模式。还充分用了gulp和webpack的搭配使用，所以总的来说还是有一些收获。另外很想说的就是sudoku这个单词，su数，doku独，小年糕知道这是一个日语单词，还是记反了，然后项目名都写错了，结果哭泣，那就先勉强将错就错，目前还是学知识更重要哈~

gulp & webpack

要说gulp和webpack的区别，这次我还是大概体验到了，写gulpfile的时候，就是写下一个个任务task，比如我给它安排解析less文件的任务，还给它安排了一个调用webpack去打包文件的任务，最后末尾添加了一个watch检测它变化的任务，以保证我修改后不用自己去重新输入命令更新。
而webpack的功能就强大很多，可以根据不同的loader去解析转化文件，然后把每个不同的模块、资源打包综合起来，还可以使用很多其他的插件来做更多有趣的事情。
这个小游戏最初版本是使用了ES6语法、Less，它用gulp来进行less处理，用gulp来调用webpack去转化ES6语法。这里就要用到一个叫做webpack-stream的包来帮助我们让gulp和webpack搭配使用。
因为视频已经稍微有点旧，babel的转化包已经升级啦~所以直接用babel-preset-env就可以了。

文件结构

我们分为逻辑+UI两个部分来完成这个游戏。
逻辑部分类：

  1. 生成类：生成一个完整的数独。
  2. 数独类：由完整数独生成缺漏的数独。
  3. 检查类：检查是否符合数独要求。
  4. 工具类：生成九宫格、获取坐标等。

UI部分类：

  1. 格子类：负责九宫格的UI。
  2. 弹出窗类：负责弹出窗选择小框框的UI。

游戏逻辑

生成乱序数组

数独内容是以二维数组进行存储的。
我们需要在81个格子里生成一个乱序的答案，默认未填充的情况下，我们都填充为0。
用一个乱序算法，把这一排的数字进行一个打乱，就能生成一行带有1-9的乱序一维数组，

Fisher–Yates乱序算法
原理是每次循环从剩下的元素里随机选取一个与自己进行交换，从而就会产生一个都是被随机过的部分，和还没有被随机的元素，代码可以按以下方法实现：

shuffle(array) {
    const endIndex = array.length - 2;
    const length = array.length;
    for(let i = 0; i <= endIndex; i++) {
        //和自己或后面的某个数进行交换
        const j = i + Math.floor(Math.random() * (length - i));
        [array[i], array[j]] = [array[j], array[i]];
    }
    return array;
}

这个乱序数组的用途是在后面进行数字填充的时候，坐标的选择。

生成完整数独

我们的填充方法是先把所有1填完，再填2，再填3…直到成为一个完整的数独。
在填充的时候，我们就必须要进行一个校验：

  1. 如果这个格子原来填过了，跳过。
  2. 用工具类检查这个格子填会产生冲突吗。

checkFillable函数就是用来检查冲突的，会产生冲突的情况就是行、列、宫格内部有一样的数字。

checkFillable(matrix, n, rowIndex, colIndex) {
    const row = matrix[rowIndex];
    const column = this.makeRow().map((v, i) => matrix[i][colIndex]);
    //获取目标所在宫格数
    const { boxIndex } = boxToolkit.convertToBoxIndex(rowIndex, colIndex);
    //获取目标所在宫格已有的所有数字
    const box = boxToolkit.getBoxCells(matrix, boxIndex);

    for(let i = 0; i < 9; i++) {
        if(row[i] === n
            || column[i] === n
            || box[i] === n)
            return false;
    }
    return true;
}

然后我们使用之前的乱序数组，来随机把1填充完，再填2….以及类推。
乱序数组供这些数字进行选择，如果不行就看下一个数组里的下一个位置能不能，若摆好了这行的这个数，我们就继续递归摆下一个。
但是可能会存在填充失败的可能，所以要在构造函数中不断的进行循环判断，直到有一个正确的数独解答出现。

生成缺损数独

首先制定一个难度系数，默认是5，然后也是按随机的方式，填充0的地方就是玩家要填空的地方。

make(level = 5) {
  //生成迷盘
  this.puzzleMatrix = this.solutionMatrix.map(row => {
      return row.map(cell => Math.random() * 9 < level ? 0 : cell);
  })
}

检查玩家答案

唔，其实我们有了生成的数独和玩家的数据，直接对比一下看是不是完全一样就行了。但是这个课程又重新按checkRows、checkCols和checkBoxes来进行检查，它这个处理效果是想达到可以告诉用户哪个空填错了。和前面的其实也类似，知晓前面的方法，这里就更加简单了，所以就不赘述了。

TypeScript

引入TypeScript要把文件改为.ts文件，创建一个tsconfig.json文件，并且安装typescript和ts-loader模块，因为游戏还引入了jQuery，为了让它能识别$符号，还要安装一个叫做@types/jquery的模块。
TypeScript和Java有很多类似点啦！所以还是挺好上手的，它用ES6的语法，但在tsconfig.json文件里面的target如果是es5，有的方法，比如Array.fill()就会识别不了，所以改为ES6啦！它的类成员必须要在前面申明，一般也有类型的要求等等。这次算是先小小接触一下，下次再继续补充吧。

其他补充

分号这个东西还是不能少，因为各种代码习惯都感觉见过，这次写了代码后发现

for(let i = 0; i <= endIndex; i++) {
    const j = i + Math.floor(Math.random() * (length - i));
    [array[i], array[j]] = [array[j], array[i]];
}

这段代码如果不加分号，就会报错。
另外当前gulp-less版本用不了3.0.4版本的Less，需要降级，可以使用npm install less@2.7.3进行安装。
