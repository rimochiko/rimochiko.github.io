CSS上下切换文字效果

今天考了密码学，可以说是凉凉了，学到的知识都没考😭，但是我还是来恢复一下心情吧。所谓的上下切换文字效果，就是，你点进来看到的这个样子，还比较常见呢，虽然是一个比较简单的效果，但是还是有一些点开始让我思考。

动画解析

这个速度的动画，人的肉眼还是比较难分清楚吧。仔细看，鼠标hover的过程其实是单数的字母和双数的字母分别从上、下进入，然后又拼凑成了新的单词。
再不行，就先只看一个字母，比如第一个B，鼠标移动进入的时候，原有的B往上走，紧接着后面又有一个B跟着上来。同理，其他字母也是一样的，只是可能方向不同。

实现

根据解析知道其实是给每个字母又附加了一个相同字母，这点用伪元素做到，然后给伪元素加上绝对定位，来控制元素的位移。
溢出的部分就使用overflow:hidden来隐藏掉，可以尝试着把代码中的这句话删除，就可以看到它们原本的移动。

抠细节

Q：这里给每个字母节点span都设置了display: inline-block，倘若注释掉这句，会发现没有动画效果，这是为什么？
A：因为动画的实现是使用了transform来实现的，CSS transform属性只对block级元素生效。

Q：这里的transform使用了如translateY(100%)，这个百分比的含义是？
A：百分号是针对自己本身的高度/宽度来计算的。也因为这样所以可以用这个类似的去设置-50%来实现元素垂直居中。

Q：比如给span添加相对定位，要不然一开始未产生效果的时候，::before元素没有被隐藏。（这里其实有个隐藏的问题BUG，待解决）
A：overflow:hidden并不隐藏所有的子元素（http://www.pc6.com/infoview/Article_51111_all.html），只有这个父亲元素的定位不是position:relative和position:absolute样式，且子元素不是absolute定位才成立。

这是我的一个测试DEMO（比较乱，噗），分别对伪元素和子元素进行了测试，目前发现两者没啥区别。
