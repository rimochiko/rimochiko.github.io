初次~Vue做项目の小结

最近这两个礼拜，我把Vue的文档浏览了一下，然后做了一些简单的例子后就直接开始动手做小项目了，为后面暑假的实训项目做个好的铺垫吧（￣▽￣）。想着确实也应该小结一下，要不然后面肯定很容易忘记，当然啦我是第一次做，所以如果有错误欢迎提出。

推荐工具

Vue中文文档
W3Cplus：Vue分类文章

入手篇

打开我们的中文文档，跟着教程一步步来安装需要的东西，就可以了。
使用vue-cli这个工具，就可以自动给我们打造一个项目结构出来，包括了webpack的配置文件呀模块的依赖呀之类的东西~

vue init webpack 项目名

初始化项目

进行初始化的时候它会给我们提问，这里先列出几个不太熟悉的选择：

? Install vue-router? (Y/n)

是否使用vue-router，因为要做的是会有“多个页面”的应用，所以肯定是yes啦~打上双引号是因为它实质上并没有跳转页面，只是网页的DOM结构随着路由地址的改变而显示不同的内容，所以这就是单页面应用，它的切换速度非常之快。

? Use ESLint to lint your code? (Y/n)

ESLint是一个帮助你检测代码规范的小工具，如果你的代码不符合规范，它就会列出一排排的警告Σ(っ °Д °;)っ（作为处女座宝宝，一个警告都容忍不下2333），使用还是好的，可以规范文件的格式，还会让你选择一种eslint preset，默认是Standard，比如我目前碰到的规范有：

  • 代码最后一行结束必须要换行
  • 使用空格来缩进，不要用tab键
  • 对象{}里面最后一个值后不能有逗号
  • 函数注意留空格：函数名 () {}
? Set up unit tests (Y/n)

根据字面意思就知道是“单元测试”，这个功能对我没什么需求，所以就是No啦。不过如果写了yes结构上就会多一个unit文件夹。

Setup e2e tests with Nightwatch?

是否使用使用Nightwatch进行E2E测试，我也是No。更多详细可看下面这篇文章：使用Nightwatch进行E2E测试

项目代码都是放在src这个文件夹下面的。打开build下的webpack.base.conf.js文件：

这里的alias是“别名”的意思，其中个我们用@这个符号来代表了src这个目录，所以我们在引入内容的时候，可以直接用@来引入src目录下的文件，这样特别方便。

一般情况下src文件夹下面有assets文件夹，用来存放图片、样式等资源。
页面的文件夹结构安排，可以分页面来分文件夹，然后各个页面下有一个存放页面内组件文件的文件夹。当然也有的是把组件抽出来作为src的一个一级目录（默认生成的模板也是这样的），这个可以按自己的需求来确定。

开始写吧

Vue文件

.js文件里面我们都是创建一个vue的实例，然后绑定dom。做大一点的应用这种方式其实挺不好操作的，所以一般都是直接创建一个.vue文件来分别写每个部分。里面的结构就是：

<template>
</template>
<script>
export default {
}
</script>
<style>
</style>

template是填写结构，script填写脚本，style填写样式。
export default的对象其实就是一个vue实例，相当于对外扩出了一个组件，因为组件可能被很多个地方使用，所以这里的data在定义的时候要返回一个对象，这样才能让每个使用它的地方数据独立：

export default {
  name: '组件名',
  data () {
    return {
      //your data
    }
  }
}

Q: 如何使用这些组件？
A：在其他地方import，然后在components中注册一下，就可以在template里面使用。

<template>
  <div>
    <home-header></home-header>
  </div>
</template>

<script>
import HomeHeader from './components/Header'
export default {
  name: 'Home',
  components: {
    HomeHeader,
  }
}
</script>

布局问题

这里主要是用了stylus来编写样式，虽然我个人觉得这个排版真的看着太难受了。
因为主要是移动端的浏览，所以选择了rem来进行尺寸的衡量。并且还有一些技巧感觉也是经常用在移动端的布局上，比如用padding-bottom来做自适应高度，还有结合用flex布局（因为之前写RN，所以很疑惑为什么不直接都全用flex，不过这样要写的语句也很繁琐，因为它不支持继承嘛Σ(っ °Д °;)っ），另外就是absolute和overflow用的很多。

引用的代码

类似于用了一些别人封装的很好的组件，或者别人的写的代码哈哈哈，比如有轮播图呀之类的。
我来总结一下自己用到的组件/外部引入的东西：
vue-awesome-swiper
是一个移动端交互的可拖动/滑动视图组件（布吉岛怎么形容更好惹…）
，因为用的地方有很多，所以都是直接在根部文件就引入了。

fontello
这里我用了一个以前没用过的fontello来做图标字体，这家的好处在于它综合了好几家免费图标字体，坏处在于引入的时候和font-awesome比起来稍微麻烦了一些。本来之前是准备用阿里的Icon-font，感觉网站做的不太好，经常没内容，要多刷新读缓存，于是果断换了体验感更好的产品哈哈哈。

模拟数据

我这次是使用了国内的eolinker来模拟测试数据，这个网站还挺好用的，完全满足个人开发的需求啦。
创建好调试的API之后，直接使用axios来获取数据（axios确实是很简便啊，跨域也没问题>_<，其实我还是很想自己手动来试试解决一下跨越的，下次试试）。

遇到的坑：

  1. JSON的严格要求：比如要用双引号把每个属性和值给包裹起来。
  2. axios的回调函数里不能直接用this来获取vue实例的东西~毕竟封装了，得重新保存一下。
