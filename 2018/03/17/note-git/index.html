学习小记

Git相关小结

前言

虽然会一些基本的Git应用，但是没有深入了解下去过，所以这又是一个慢慢更新和不断学习以及小结的坑。主要是学习了廖雪峰老师的Git教程，确实是简单易懂。

基本知识

Q：什么是Git？

一款分布式版本控制系统，集中式版本控制系统必须联网才能工作。分布式版本控制系统根本没有“中央服务器”，每个人的电脑上都是一个完整的版本库。
注意：Git管理的是修改。

Q：版本库里有什么？

工作区有一个隐藏目录.git，是Git的版本库，存了很多东西，其中最重要的就是称为stage（或者叫index）的暂存区，还有Git为我们自动创建的第一个分支master，以及指向master的一个指针叫HEAD。

Q: 一般的分支类型有哪些？

master分支是主分支，因此要时刻与远程同步；
dev分支是开发分支，团队所有成员都需要在上面工作，所以也需要与远程同步；
bug分支只用于在本地修复bug，就没必要推到远程了，除非老板要看看你每周到底修复了几个bug；
feature分支是否推到远程，取决于你是否和你的小伙伴合作在上面开发。

常用命令

git init
创建版本库。使用后目录下会多一个.git的目录，用于跟踪管理版本库（Repository）。

git add
添加文件到仓库。git add 文件名。

git commit
提交到仓库，一次性提交多个，选项-m “说明”，来添加改动的说明。

git diff
查看改动。

git status
显示仓库的版本状态。

git log
历史记录
git log --graph用于查看历史合并图。

git reflog
记录历史操作

git reset

git reset --hard HEAD^

可以用于退回到上一个版本。

git checkout
撤销修改，如:git checkout -- readme.txt。
切换分支，如:git checkout dev。
创建并且切换分支，如:git chekoout -b dev。

git rm
删除文件，如：git rm test.txt。

git branch
创建分支。后面加分支名。
git branch -d 分支名表示删除分支。
git branch -D 分支名表示强行删除分支。

git merge
合并分支。如：git merge dev。Fast-forward是指是快进模式，直接把master指向dev的当前提交。不是每次合并都能Fast-forward。
–no-ff参数用于禁止快进模式。

git push
推送分支。
git push origin :refs/tags/标签名，删除远程标签。

git clone
克隆仓库。

git stash
如果有两个地方改动了，但是只想提交其中一个，可以把当前工作现场保存起来，等恢复现场后继续工作。
git stash list查看所有保存的工作现场。
git stash apply恢复工作现场。
git stash drop删除保存工作现场。
git stash pop恢复并删除。

git remote
查看远程库的信息。
git remote -v可以显示更加详细的信息。

git tag
添加/查看标签。
git tag 标签名 commitID可以给指定commit添加标签。
git tag -d 标签名删除标签名。

git show
git show 标签名可以查看标签信息。
git tag -a 标签名 -m "..."指定标签信息。
git tag -s 标签名 -m "..."用PGP签名标签；
