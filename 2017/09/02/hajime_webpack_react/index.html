小结~webpack+react.js踩坑

之前跟着慕课网老师的课程做东西，结果…到处都是坑，首先是因为那个老师真的跳着讲呀莫名其妙很多东西，也不知道都有什么用，还有一个就是技术更新太快了，很多用法都落伍了，这里用到的技术/小工具有webpack、react-hot-loader、react.js等。

so，let’s begin…
首先是在写webpack.config.js这个配置文件的时候，老师直接写了一个相对路径，在后版本的webpack中发生了改变，需要用下面的函数来连接成绝对路径。

path.join([path1][, path2][, …])

用于连接路径。该方法的主要用途在于，会正确使用当前系统的路径分隔符，Unix系统是，Windows系统是\。

path.resolve([from …], to)

将 to 参数解析为绝对路径。
随后就是解析css呀之类的用到的loader，都要npm install去安装~否则就会出现找不到XX模块的标识，直接安装就好了。
这个小东西用了less，css等，要安装less-loader,css-loader,style-loader，此外新版本中装less-loader不会把less模块一起下载，所以还要再安装less模块。

安装react的时候要写npm install react react-dom –save-dev，
因为现在的包已经一分为二了，所以要装两个，–save-dev则是可以把这个依赖信息存入package.json里。

还有解析react.js的时候，如果我们想使用es6的语法，就必须要把es6语法转换为es5语法，而babel就帮助我们解决了这个问题，所以我们要装：npm install babel-preset-es2015 babel-preset-react –save-dev

{
        test:/\.js$/,
        exclude: /node_modules/,
        loader:"babel-loader",
        query:
        {
          "presets": [
              "react",
              "es2015",
          ]

        }
}

当你写上这里到配置文件的时候，打包可能会出错，语法没问题的话，那就应该是你缺少了组件。(2018/03/12后续备注：如今已经把es2015升级成统一的babel-preset-env了，配置也就是把es015改成env就行)

随后老师还用到了webpack-dev-server和react-hot-loader，使用这个控件可以做到实时更新状态~还需要写一个server.js文件来运行（所以前提是你装好了Node.js），但是这个过程中我的出了好多错误=_=。
首先是要安装好这个模块（全局安装），再修改entry配置：

'webpack-dev-server/client?http://localhost:3000', // WebpackDevServer的主机和端口号
'webpack/hot/only-dev-server', // "only" 阻止重新载入报错
APP_PATH // 你应用的开始入口

这是一般的server.js（举个例子）

var webpack = require('webpack');
var WebpackDevServer = require('webpack-dev-server');
var config = require('./webpack.config');

new WebpackDevServer(webpack(config), {
  publicPath: config.output.publicPath,
  hot: true,
  historyApiFallback: true
}).listen(3000, 'localhost', function (err, result) {
  if (err) {
    return console.log(err);
  }

  console.log('Listening at http://localhost:3000/');
});

在package.json文件里面修改 “scripts”: {“start”: “node server.js”}，
代表运行这个的时候需要启动服务。

然后回到config.js，添加插件

plugins: [
    new webpack.HotModuleReplacementPlugin()
]

随后的在react-hot-loader官网都还写的挺清楚的：http://gaearon.github.io/react-hot-loader/getstarted/
最后还要自动生成HTML，要安装html-webpack-plugin。最后在插件那里相关配置就好了。啊，没养成边做边做笔记的习惯，结果自己现在也忘记有什么错误了QAQ，下次一定要QAQ。
